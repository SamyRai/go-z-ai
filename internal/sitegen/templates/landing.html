Install

# Install the CLI
go install {{ .Site.Module }}@latest

# The binary is named go-z-ai — symlink or rename to zai-client
ln -s "$(go env GOPATH)/bin/go-z-ai" "$(go env GOPATH)/bin/zai-client"

# Configure
export ZAI_API_KEY=your_api_key_here

# Use it
zai-client chat create "Explain goroutines" --stream

Or use it as a Go library — see the Library Guide.

Quick example

{{- /* godoccomment: the literal {{ below is escaped via a template string to avoid clashing with Go template syntax. */ -}} 
package main

import (
    "context"
    "fmt"
    "github.com/{{ .Site.Owner }}/{{ .Site.Repo }}/pkg/client"
)

func main() {
    c, _ := client.NewClientFromEnv()
    resp, _ := c.Chat().Create(context.Background(), client.ChatRequest{
        Model:    "glm-5.2",
        Messages: []client.Message{{ "{{" }}Role: "user", Content: "Hello"{{ "}}" }},
    })
    fmt.Println(resp.Choices[0].Message.Content)
}

What's covered

This single binary gives you CLI, library, and TUI access to every Z.AI model surface:

  • Chat — streaming, structured output (JSON Schema), deep thinking, function/tool calling, vision
  • Anthropic-compatible endpoint — the same surface Claude Code/Cursor hit on a GLM Coding Plan
  • Media — image, video, audio transcription, TTS, voice cloning
  • Document understanding — layout OCR, handwriting OCR, document parser for RAG
  • Retrieval — embeddings, rerank, web search/reader/tokenizer tools
  • Agents, moderations, batch, files
  • GLM Coding Plan — quota/usage, multi-account, wiring Claude Code/Cursor/...
  • Full-screen TUI — zai-client tui

How it relates to the official SDKs

Z.AI publishes official SDKs for Python, Node, and Java. There is no official Go SDK — go-z-ai fills that gap, and layers a CLI, a TUI, regional gateway switching, and GLM Coding Plan multi-account management on top of the same API surface.
